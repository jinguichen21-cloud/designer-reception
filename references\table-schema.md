# AI 表格存储结构

## Base: 设计团队管理

由 designer-reception 初始化创建，包含 3 张表。

### 表 1: 渠道管理

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| 渠道名称 | text | 是 | 平台名称(xiaohongshu/douyin/weibo等) |
| 状态 | select | 是 | 运行中/已停用 |
| 登录账号 | text | 否 | 平台用户名 |
| 监控类型 | text | 否 | comments,likes,follows |
| 轮询间隔 | number | 否 | 秒，默认 60 |
| 创建时间 | date | 是 | 渠道添加时间 |

### 表 2: 接待任务

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| 任务名称 | text | 是 | 接待任务描述 |
| 渠道 | text | 是 | 关联渠道 |
| 状态 | select | 是 | 运行中/已暂停/已完成 |
| 开始时间 | date | 是 | 任务启动时间 |

### 表 3: 咨询记录

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|:----:|------|
| 客户名称 | text | 是 | 客户昵称或名称 |
| 渠道 | text | 是 | 来源渠道 |
| 咨询内容 | text | 是 | 客户咨询的原始内容 |
| 回复内容 | text | 否 | AI建议 + 用户确认后的回复 |
| 状态 | select | 是 | 待跟进/已回复/已转Brief/已关闭 |
| Brief链接 | url | 否 | 关联的Brief文档链接 |
| 创建时间 | date | 是 | 咨询时间 |
